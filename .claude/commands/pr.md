# Auto PR — GitHub Pull Request 생성

현재 브랜치의 커밋을 분석하여 PR 제목과 본문을 자동으로 작성하고, 사용자가 지정한 타겟 브랜치로 GitHub PR을 생성합니다.

## 실행 절차

### 1. 타겟 브랜치 확인

`$ARGUMENTS`에 타겟 브랜치가 지정되어 있으면 그대로 사용합니다.
지정되어 있지 않으면 **사용자에게 타겟 브랜치를 입력받습니다** (예: `master`, `main`, `dev`).

### 2. 현재 상태 파악 (병렬 실행)

- `git branch --show-current` — 현재 브랜치 확인
- `git log <타겟브랜치>...HEAD --oneline` — 타겟 브랜치 대비 커밋 목록 확인
- `git diff <타겟브랜치>...HEAD --stat` — 변경 파일 통계 확인
- `gh pr list --head $(git branch --show-current)` — 이미 열린 PR 여부 확인

이미 열린 PR이 있으면 사용자에게 알리고 중단합니다.

### 3. 원격 브랜치 푸시 확인

현재 브랜치가 원격에 없거나 뒤처져 있으면:
- `git push -u origin <현재브랜치>` 실행 후 계속 진행

### 4. PR 내용 분석 및 초안 작성

커밋 로그와 diff를 분석하여 아래 형식으로 PR 초안을 작성합니다.

**PR 제목 규칙**
- 70자 이하
- 변경의 핵심을 한 줄로 요약
- Conventional Commits 스타일 사용 (예: `feat(blog): 게시글 목록 페이지 추가`)

**PR 본문 형식**
```
## 개요
<변경 내용을 2-4문장으로 요약>

## 변경 사항
- <주요 변경 항목 불릿>
- ...

## 테스트 방법
- [ ] <테스트 체크리스트>
- ...
```

### 5. 사용자 확인

작성한 PR 제목과 본문을 사용자에게 보여주고 확인을 받습니다.
수정이 필요하면 수정 후 재확인합니다.

### 6. PR 생성

```bash
gh pr create \
  --title "<PR 제목>" \
  --body "<PR 본문>" \
  --base <타겟브랜치> \
  --head <현재브랜치>
```

### 7. 결과 출력

생성된 PR URL을 출력합니다.

---

## 주의사항

- 커밋이 없거나 타겟 브랜치와 동일한 상태이면 PR 생성 없이 종료합니다
- `gh auth status`가 실패하면 `gh auth login` 안내 후 중단합니다
- PR 제목과 본문은 반드시 사용자 확인 후 생성합니다
